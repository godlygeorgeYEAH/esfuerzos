# Paso 6 — Notificación de coincidencia (send_match_notification)

Añade la función de notificación WhatsApp para cuando el sistema detecta una posible coincidencia entre un reporte de desaparecido y uno de persona encontrada.

---

## Archivos modificados

**`app/services/waha.py`** — nueva función `send_match_notification`:

```python
await send_match_notification(
    phone="584121234567",
    missing_name="María García",
    found_location="Refugio Cumaná Norte",
    found_state="alive",          # alive | injured | deceased | unknown
    coordinator_contact="@coord_reune",
    session="default",
)
```

Mensaje generado:
```
🔔 *Posible coincidencia para María García*

Encontramos a una persona con vida en:
📍 Refugio Cumaná Norte
📞 Contacto de verificación: @coord_reune

⚠️ Esta es una coincidencia *preliminar*.
Un coordinador deberá verificarla antes de confirmar.

Responde *CONFIRMAR* si deseas que un coordinador te contacte.
```

Reutiliza `send_message` internamente — respeta `waha_api_key` y `waha_session`.

---

## Pendientes

### Motor de matching (no implementado)

`send_match_notification` solo envía el mensaje. El servicio que detecta coincidencias entre reportes `missing` y `found` no existe aún. Requiere:
- Comparar `full_name`, `age`, `last_seen_location` entre reportes
- Umbral de similitud configurable
- Tarea periódica o trigger al crear un reporte `found`

### Flujo `found` (rescatista / hospital)

No hay flujo de intake para rescatistas ni hospitales. Sin reportes `found` en DB, el matching nunca se activa. Bloqueado por la implementación de `guia_rescatista` y `guia_hospital`.

### Respuesta a CONFIRMAR

El usuario puede responder `CONFIRMAR` tras recibir la notificación. El bot actualmente no tiene un nodo que maneje esa keyword — queda en fallback.

### Teléfono del reportero

`reporter_wa_hash` en `Report` es un hash SHA-256 — no se puede revertir al número original para enviar la notificación. El servicio de matching debe obtener el teléfono desde `Conversacion.client_phone` cruzando por hash, o almacenar el `waha_chat_id` en el reporte.

### Pruebas

No hay curl de prueba directo para esta función porque depende del motor de matching. Probar manualmente importando y llamando la función desde un script con un WAHA activo.
