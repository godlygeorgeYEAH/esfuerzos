# Paso 4 — Sesión multi-foto con keyword "listo"

El usuario acumula fotos y escribe *listo* para cerrar la sesión y avanzar. El bot también avanza automáticamente si se alcanzan las 5 fotos. Los reportes sin fotos son válidos: escribir *listo* sin haber enviado fotos avanza igualmente.

---

## Archivos modificados

**`app/config.py`** — Dos variables de configuración de fotos:

| Variable | Default | Descripción |
|---|---|---|
| `photo_max_count` | `5` | Fotos máximas por sesión; al alcanzarlo avanza inmediatamente |
| `photo_storage_path` | `media/photos` | Directorio local de descarga |

> `photo_ttl_seconds` existe en config pero ya no controla el avance del flujo.

**`app/bot/orchestrator.py`** — Tres métodos:

- `_handle_pedir_foto`: intercepta toda interacción en nodo `pedir_foto`. Si llega `media_url` descarga la foto y acumula en `context["pending_photos"]`. Si llega texto, evalúa si es `"listo"`.
- `_advance_from_foto`: navega a `notas_adicionales`, genera respuesta del template.
- `_download_photo`: GET httpx al `media_url`, detecta ext por Content-Type, guarda en `{photo_storage_path}/{conversation_id}_{index}.{ext}`.

---

## Lógica de avance

```
Foto recibida
  → descargar + guardar en pending_photos
  → count >= photo_max_count → avanzar (motivo: max_alcanzado)
  → si no → "N/max recibidas. Escribe *listo* cuando termines."

Texto recibido == "listo"
  → avanzar (motivo: listo) — con 0 o más fotos

Texto recibido != "listo"
  → "Tienes N/max foto(s). Puedes enviar más o escribe *listo*."
```

Las fotos se almacenan en `context["pending_photos"]` como lista de `{media_url, local_path, received_at}`. El `report_id` se asigna en el Paso 5.

## Cómo probar el Paso 4

```bash
BASE="http://localhost:8000/webhook/waha"
PHONE="584121234567@c.us"

# 1. Hola → bienvenida
curl -s -X POST $BASE -H "Content-Type: application/json" \
  -d "{\"session\":\"default\",\"event\":\"message\",\"id\":\"t1\",\"payload\":{\"from\":\"$PHONE\",\"fromMe\":false,\"body\":\"hola\",\"hasMedia\":false}}"

# 2. Seleccionar familiar
curl -s -X POST $BASE -H "Content-Type: application/json" \
  -d "{\"session\":\"default\",\"event\":\"message\",\"id\":\"t2\",\"payload\":{\"from\":\"$PHONE\",\"fromMe\":false,\"body\":\"1\",\"hasMedia\":false}}"

# 3. Datos de la persona
curl -s -X POST $BASE -H "Content-Type: application/json" \
  -d "{\"session\":\"default\",\"event\":\"message\",\"id\":\"t3\",\"payload\":{\"from\":\"$PHONE\",\"fromMe\":false,\"body\":\"María García, femenino, 34, Cumaná centro\",\"hasMedia\":false}}"
# Esperado: pedir_foto

# 4. Primera foto
curl -s -X POST $BASE -H "Content-Type: application/json" \
  -d "{\"session\":\"default\",\"event\":\"message\",\"id\":\"t4\",\"payload\":{\"from\":\"$PHONE\",\"fromMe\":false,\"body\":\"\",\"hasMedia\":true,\"mediaUrl\":\"http://waha/media/foto1.jpg\"}}"
# Esperado: "📸 Imagen recibida (1/5). Escribe *listo* cuando termines."

# 5. Segunda foto
curl -s -X POST $BASE -H "Content-Type: application/json" \
  -d "{\"session\":\"default\",\"event\":\"message\",\"id\":\"t5\",\"payload\":{\"from\":\"$PHONE\",\"fromMe\":false,\"body\":\"\",\"hasMedia\":true,\"mediaUrl\":\"http://waha/media/foto2.jpg\"}}"
# Esperado: "📸 Imagen recibida (2/5). Escribe *listo* cuando termines."

# 6. Texto que no es "listo" → debe quedarse en pedir_foto
curl -s -X POST $BASE -H "Content-Type: application/json" \
  -d "{\"session\":\"default\",\"event\":\"message\",\"id\":\"t6\",\"payload\":{\"from\":\"$PHONE\",\"fromMe\":false,\"body\":\"ya terminé\",\"hasMedia\":false}}"
# Esperado: "⏳ Tienes 2/5 foto(s). Puedes enviar más o escribe *listo*."

# 7. "listo" → avanza a notas_adicionales
curl -s -X POST $BASE -H "Content-Type: application/json" \
  -d "{\"session\":\"default\",\"event\":\"message\",\"id\":\"t7\",\"payload\":{\"from\":\"$PHONE\",\"fromMe\":false,\"body\":\"listo\",\"hasMedia\":false}}"
# Esperado: notas_adicionales
```

### Probar reporte sin fotos

```bash
# Tras llegar a pedir_foto, escribir "listo" sin haber enviado fotos
curl -s -X POST $BASE -H "Content-Type: application/json" \
  -d "{\"session\":\"default\",\"event\":\"message\",\"id\":\"t8\",\"payload\":{\"from\":\"$PHONE\",\"fromMe\":false,\"body\":\"listo\",\"hasMedia\":false}}"
# Esperado: avanza a notas_adicionales (0 fotos es válido)
```

### Verificar fotos en contexto

```bash
python - <<'EOF'
import json
from app.database import SessionLocal
from app.models.bot import Conversacion
db = SessionLocal()
conv = db.query(Conversacion).filter(Conversacion.client_phone == "584121234567").first()
ctx = json.loads(conv.context or "{}")
print(f"Nodo actual: {conv.current_node_key}")
print(f"Fotos pendientes: {len(ctx.get('pending_photos', []))}")
for p in ctx.get("pending_photos", []):
    print(f"  - {p['local_path']} ({p['received_at']})")
db.close()
EOF
```

## Pendientes

### Descarga real de WAHA

`_download_photo` hace GET directo al `mediaUrl`. En producción WAHA requiere el header `X-Api-Key` para descargar media si la auth está habilitada.

### Conversaciones varadas

Si el usuario llega a `pedir_foto` y nunca vuelve a escribir, la conversación queda en ese nodo indefinidamente. Para producción: tarea periódica que detecte conversaciones inactivas y las avance o archive.
