Eliminar todo el código food-delivery del módulo portado para dejarlo neutro antes de construir el intake de crisis.

| Archivo | Razón |
|---|---|
| `app/bot/abc_layer.py` | Lógica de respuestas automáticas para pedidos |
| `app/services/storage.py` | Almacenamiento S3 de comprobantes de pago |

## Archivos modificados

**`app/bot/flow_engine.py`** — Eliminado `from app.models.menu import Articulo` (import bloqueante en arranque, PROBLEMATICAS §7). Eliminados todos los branches commerce de `_generate_response()` (`ver_menu`, `pedido_recibido`, `info_negocio`, etc.) y el método `_build_payment_method_messages()`.

**`app/bot/orchestrator.py`** — Eliminados: instanciación de `ABCLayer`, `_handle_comprobante()`, `_handle_texto_en_espera_comprobante()`, shortcircuit de CTA, Step 12 (ABC Layer), bloque de `crear_notificacion` en escalado humano.

**`app/bot/response_generator.py`** — Eliminados: `from app.models.menu import Articulo`, `render_articulo_list`, `render_payment_methods`, `_get_modalidades()`, todos los métodos `_prompt_*` gastronómicos. Simplificado a `_load_context()` genérico y `_build_node_prompt()`.

**`app/bot/template_renderer.py`** — Eliminados `render_articulo_list()` y `render_payment_methods()`.

**`app/bot/analytics_logger.py`** — Eliminado parámetro `abc_applied` de `log_response()`.

**`app/routers/webhook.py`** — Eliminado bloque conductor (PROBLEMATICAS §5): imports de `Conductor`, `es_respuesta_conductor`, `procesar_respuesta_conductor`, `normalize_phone`, `get_or_create_cliente` y el branch de procesamiento.

**`app/models/negocio.py`** — Eliminados campos: `metodos_pago`, `datos_pago`, `delivery_enabled`, `retiro_enabled`, `negocio_lat`, `negocio_lng`, `direccion`.

**`app/config.py`** — Eliminadas variables: `webapp_base_url`, `media_base_url`, `storage_backend`, `s3_*`.

**`app/main.py`** — Eliminados: `StaticFiles` mount de `/media`, creación de subdirectorios de media. Corregido bug CORS (`allow_origins=[]` en producción → `["*"]`).

## Resultado

El módulo arranca sin errores de import. Quedan 27 archivos Python (de 37). El flujo conversacional base (orquestador, motor de decisión, detector de intención, motor de flujo) está intacto y listo para recibir los nodos de intake de crisis.

## Cómo probar el Paso 1

### Smoke test — imports sin errores

```bash
cd esfuerzos/modulos/migration_prox
DATABASE_URL="sqlite:///./test.db" python -c "
from app.main import app
print('OK — módulo arranca sin errores de import')
"
```

Confirma que el import bloqueante original (`Articulo`) y todos los módulos eliminados ya no son referenciados.

### Integration test — webhook end-to-end

**1. Crear `.env` mínimo:**
```env
DATABASE_URL=sqlite:///./test.db
WAHA_URL=http://localhost:3000
DEEPSEEK_API_KEY=
ENVIRONMENT=development
```

**2. Crear tablas y negocio de prueba:**
```bash
python - <<'EOF'
from app.database import Base, engine
from app.models import bot, negocio
Base.metadata.create_all(engine)

from app.database import SessionLocal
from app.models.negocio import Negocio
db = SessionLocal()
db.add(Negocio(nombre="Reúne Test", slug="reune-test", waha_session="default", is_active=True))
db.commit()
db.close()
print("Listo")
EOF
```

**3. Arrancar:**
```bash
uvicorn app.main:app --reload
```

**4. Enviar mensaje de prueba:**
```bash
curl -s -X POST http://localhost:8000/webhook/waha \
  -H "Content-Type: application/json" \
  -d '{
    "session": "default",
    "event": "message",
    "payload": {
      "from": "584121234567@c.us",
      "fromMe": false,
      "body": "Hola",
      "hasMedia": false
    }
  }' | python -m json.tool
```

**Respuesta esperada:** `{"status": "processed", "sent": false}`

El pipeline se ejecuta completo (logs visibles en consola). El `send` es `false` porque no hay servidor WAHA corriendo — eso es normal en este punto.
