# Paso 2 — Modelos de datos para intake de crisis

Crear los modelos SQLAlchemy `Report` y `Photo` que almacenarán los reportes de personas desaparecidas y encontradas recibidos por WhatsApp.

## Archivo creado

`app/models/reporte.py`

## Modelos

### `Report`

| Campo | Tipo | Descripción |
|---|---|---|
| `id` | `int PK` | Autoincremental |
| `kind` | `str(10)` | `'missing'` (familiar busca) o `'found'` (rescatista reporta) |
| `full_name` | `text` | Nombre completo de la persona |
| `age` | `int` | Edad aproximada |
| `last_seen_location` | `text` | Última ubicación conocida |
| `distinguishing_marks` | `text` | Señas particulares |
| `clothing` | `text` | Descripción de ropa |
| `person_state` | `str(20)` | `alive` / `injured` / `deceased` / `unknown` |
| `reporter_wa_hash` | `str(64)` | SHA-256 del número WhatsApp del reportante |
| `consent` | `bool` | Consentimiento explícito registrado durante el intake |
| `source` | `str(50)` | Origen del reporte (default: `'whatsapp'`) |
| `created_at` | `timestamptz` | Timestamp de creación |
| `expires_at` | `timestamptz` | Expiración opcional del reporte |

### `Photo`

| Campo | Tipo | Descripción |
|---|---|---|
| `id` | `int PK` | Autoincremental |
| `report_id` | `int FK` | Referencia a `reports.id` (CASCADE delete) |
| `media_url` | `text` | URL de la foto servida por WAHA |
| `quality_ok` | `bool` | Gate de calidad facial (null hasta que CompreFace evalúe) |
| `created_at` | `timestamptz` | Timestamp de creación |

## Decisiones de diseño

- **Sin `text_embedding` ni `face_embedding`:** las columnas `vector(1536)` y `vector(512)` del PRD se agregan cuando se integre Supabase/pgvector. No se incluyen aquí para no bloquear el desarrollo del canal WhatsApp.
- **`reporter_wa_hash` en lugar del número crudo:** el número de teléfono nunca se guarda en texto plano. El hash SHA-256 permite recontactar al reportante sin exponer su número.
- **`quality_ok` en `Photo`:** se setea a `True`/`False` cuando CompreFace evalúe la foto. Mientras tanto es `null` (foto recibida, pendiente de evaluación).
- **Estilo `mapped_column`:** SQLAlchemy 2.0, consistente con `negocio.py`.

## Cómo probar el Paso 2

### Verificar que las tablas se crean correctamente

```bash
cd esfuerzos/modulos/migration_prox
DATABASE_URL="sqlite:///./test.db" python - <<'EOF'
from app.database import Base, engine
from app.models import bot, negocio
from app.models.reporte import Report, Photo

Base.metadata.create_all(engine)
print("Tablas creadas:", list(Base.metadata.tables.keys()))
EOF
```

**Salida esperada** (incluye las nuevas):
```
Tablas creadas: [..., 'reports', 'photos']
```

### Insertar y leer un reporte de prueba

```bash
DATABASE_URL="sqlite:///./test.db" python - <<'EOF'
from app.database import SessionLocal
from app.models.reporte import Report, Photo

db = SessionLocal()

reporte = Report(
    kind="missing",
    full_name="María González",
    age=34,
    last_seen_location="Cumaná, Sucre",
    distinguishing_marks="Lunar en mejilla derecha",
    clothing="Blusa azul, jeans",
    reporter_wa_hash="a" * 64,  # hash ficticio para la prueba
    consent=True,
)
db.add(reporte)
db.flush()

foto = Photo(report_id=reporte.id, media_url="http://waha/media/abc123.jpg")
db.add(foto)
db.commit()

r = db.query(Report).first()
print(f"Report: id={r.id}, kind={r.kind}, name={r.full_name}")
print(f"Photos: {db.query(Photo).filter(Photo.report_id == r.id).count()}")
db.close()
EOF
```

**Salida esperada:**
```
Report: id=1, kind=missing, name=María González
Photos: 1
```
